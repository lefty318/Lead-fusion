# Getting Started - Step by Step

Follow these steps to get your OmniLead application up and running.

## üöÄ Quick Setup (Automated)

### Backend Setup

1. **Run the setup script:**
   ```bash
   cd backend
   python setup.py
   ```

   This will:
   - Create a `.env` file with a secure SECRET_KEY
   - Check dependencies
   - Verify database connection
   - Check migration status

2. **Install dependencies (if needed):**
   ```bash
   pip install -r requirements.txt
   ```

3. **Update your `.env` file:**
   - Edit `backend/.env`
   - Update `DATABASE_URL` with your PostgreSQL credentials
   - Add `OPENAI_API_KEY` if using AI features

4. **Set up the database:**
   ```bash
   # Create database (in PostgreSQL)
   CREATE DATABASE omnilead;
   
   # Create and apply migrations
   alembic revision --autogenerate -m "Initial migration"
   alembic upgrade head
   ```

5. **Start the backend:**
   ```bash
   python -m uvicorn app.main:socket_app --reload
   ```

### Frontend Setup

1. **Run the setup script:**
   ```bash
   cd frontend
   node setup.js
   ```

2. **Install dependencies:**
   ```bash
   npm install
   ```

3. **Start the frontend:**
   ```bash
   npm start
   ```

2. **Frontend:**

## üêõ Troubleshooting

### Backend Issues

**Dependencies not found:**
```bash
# Activate virtual environment first
cd backend
venv\Scripts\activate  # Windows
source venv/bin/activate  # macOS/Linux
pip install -r requirements.txt
```

**Database connection error:**
- Verify PostgreSQL is running
- Check `DATABASE_URL` in `.env` file
- Ensure database exists: `CREATE DATABASE omnilead;`

**Migration errors:**
- Review migration files in `alembic/versions/`
- Check database connection
- Try: `alembic upgrade head --sql` to see SQL without executing

### Frontend Issues

**Dependencies not found:**
```bash
cd frontend
npm install
```

**Port already in use:**
- React will prompt to use a different port
- Or change port in `package.json` scripts

**API connection errors:**
- Verify backend is running on port 8000
- Check `REACT_APP_API_URL` in `frontend/.env`
- Check CORS settings in `backend/app/main.py`

## üéØ Production Deployment

When ready for production:

1. Set `ENVIRONMENT=production` in `.env`
2. Use a secure `SECRET_KEY` (already generated by setup script)
3. Set `DEBUG=false`
4. Configure production database
5. Set up reverse proxy (nginx)
6. Enable HTTPS
7. Set up monitoring

See **[SETUP_CHECKLIST.md](SETUP_CHECKLIST.md)** for the complete production checklist.
